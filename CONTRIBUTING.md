# Contributing to Case011

Thank you for your interest in contributing to Case011! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Bugs
- Check if the bug has already been reported in Issues
- Use the bug report template
- Include steps to reproduce, expected behavior, and actual behavior
- Add screenshots if applicable

### Suggesting Features
- Check if the feature has already been suggested
- Clearly describe the feature and its benefits
- Explain how it fits with the project's goals

### Code Contributions

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Follow the existing code style
   - Add comments for complex logic
   - Test your changes thoroughly
4. **Commit your changes**
   ```bash
   git commit -m "Add: brief description of changes"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Create a Pull Request**

## Code Style Guidelines

- Use meaningful variable and function names
- Add JSDoc comments for functions
- Keep functions focused and small
- Follow the existing naming conventions
- Maintain the retro-detective aesthetic

## Adding New Cases

When adding new case seeds to `caseGenerator.js`:
- Ensure the case is based on real events
- Include proper Wikipedia links
- Add at least 3 relevant artifacts
- Write an engaging briefing
- Test the case thoroughly

## Questions?

Feel free to open an issue for any questions or discussions!
